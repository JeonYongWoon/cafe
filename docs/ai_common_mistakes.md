# AI 실수 사전 (AI Common Mistakes)

본 문서는 AI 코딩 에이전트들이 코드 작성 및 리뷰 단계에서 반복적으로 저지르기 쉬운 실수들을 기록하여 미연에 예방하기 위한 체크리스트입니다. 구현 및 리뷰 에이전트는 작업을 시작하기 전 본 문서를 필독해야 합니다.

## 1. 롬복(Lombok) 사용 규칙 위반
- **실수 패턴**: 엔티티 클래스 상단에 단순 편리성을 이유로 `@Setter` 혹은 `@Data`를 붙이는 것.
- **발생 오류**: 전역적인 불변성 파괴 및 JPA 양방향 연관관계에 의한 무한 순환 참조(StackOverflowError) 발생.
- **대비책**: 엔티티에는 `@Getter`만 적용하고, 값 변경은 의미가 구체적인 비즈니스 메서드로 처리할 것.

## 2. Bounded Context 의존성 규칙 위반
- **실수 패턴**: 타 도메인 엔티티를 `@ManyToOne` 등으로 직접 객체 참조 설정하거나 타 도메인 핵심 서비스를 직접 주입(`@Autowired`)하여 사용하는 것.
- **발생 오류**: 도메인 간의 강한 결합으로 인해 독립적인 배포가 불가능해지고 아키텍처 의존성이 엉킴.
- **대비책**: 식별자 직접 참조(Long targetId 형식)를 기본으로 하며, 다중 도메인 조율이 필요한 경우 퍼사드(Facade) 계층을 명확하게 경유할 것.

## 3. 예외 처리 규격 및 에러 코드 미준수
- **실수 패턴**: 임의로 `RuntimeException`을 직접 발생시키거나 공통 응답 JSON 규격을 무시하고 커스텀 포맷을 임의 작성하는 것.
- **발생 오류**: 프론트엔드 및 API 연동 대상 시스템과의 데이터 정합성 실패 및 에러 트래킹 불가능.
- **대비책**: [CONVENTION.md](file:///Users/t2026-m0045/Downloads/cafe/docs/CONVENTION.md)에 지정된 응답 JSON 규격과 도메인별 에러 코드 명칭(예: `POINT_INSUFFICIENT`)을 준수할 것.

## 4. 동시성 제어 부재 및 단순 Mock 테스트
- **실수 패턴**: 포인트 충전 등 동시 요청이 발생할 수 있는 코드에 단순 Mocking 단위 테스트만 작성하고 넘어가는 것.
- **발생 오류**: 실서버 운영 중 다중 스레드 상황에서 데이터 무결성이 깨져 잔액이 꼬이는 장애 발생.
- **대비책**: 동시성 제어 기술(비관적 락 등)을 반드시 적용하고, 실제 DB 트랜잭션을 수반하는 통합 테스트 코드를 구현해 정상 검증할 것.
