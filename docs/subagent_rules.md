# 서브에이전트 규칙 (Subagent Rules)

본 문서는 구현, 검증, QA, 리뷰를 담당하는 개별 서브에이전트들의 정의와 행동 규칙을 명시합니다.

## 1. 구현 에이전트 (Implement Agent)
- **역할**: 오케스트라 에이전트의 세부 기획에 맞춰 자바 및 데이터베이스 영속 계층 등의 기능 코드를 작성합니다.
- **테스트 코드 작성 요건**:
  - **고위험 작업**: `@SpringBootTest`를 활용하여 실제 데이터베이스 트랜잭션의 커밋/롤백, 외래 키 제약 조건, 동시성 이슈를 다루는 통합 테스트를 필수로 작성해야 합니다.
  - **중위험 작업**: 컨트롤러 슬라이스 테스트(`@WebMvcTest`) 및 데이터 계층 슬라이스 테스트(`@DataJpaTest`)를 작성합니다.
  - **저위험 작업**: 단순 단위 테스트 클래스를 추가합니다.

## 2. 검증 에이전트 (Verify Agent)
- **역할**: 빌드 도구(`gradlew test` 등)를 구동하여 컴파일과 테스트 통과 여부를 직접 확인하고, 검증 운영 통계를 로깅합니다.
- **검증 대상 구체화**:
  - 오케스트라 에이전트가 설정한 위험도 지침(고/중/저)에 맞춰 합당한 테스트 코드가 포함되었는지 코드 수준에서 검수합니다. 고위험 이슈인데 통합 테스트가 없거나 `@MockBean`으로만 채워진 경우 검증을 반려 처리합니다.
  - 빌드 검증이 완료되면, 해당 검증 세션 정보(사용된 에이전트 수 포함)를 로그 파일에 기록할 수 있도록 리뷰 에이전트에 이관합니다.

## 3. QA 에이전트 (QA Agent)
- **역할**: 비즈니스 요구사항 및 API 예외 처리 규격이 만족되었는지 시나리오 기반으로 확인합니다.
- **지침**:
  - 실패 케이스(예: 포인트 부족 시 `POINT_INSUFFICIENT` 에러 코드 반환 및 HTTP Status 400 Bad Request 확인)가 Mock 또는 실제 테스트 내에서 완벽하게 처리되는지 확인합니다.

## 4. 리뷰 에이전트 (Review Agent)
- **역할**: 구현, 검증, QA가 완료된 작업(PR 혹은 변경 사항)에 대해 종합 검토 후 **승인(Approve)** 또는 **거절(Reject)**을 최종 결정합니다.
- **지침**:
  - 리뷰 진행 시 피드백 강도 표기법(P1 ~ P5)을 준수하여 리뷰 코멘트를 작성합니다.
  - 사용자가 작업 과정에서 실수한 내역(잘못 설계된 API 경로, 컨벤션 위반 등) 및 오케스트레이션 단계에서 사용된 총 에이전트 인원수 정보를 포함해 공식 로그 파일로 기록을 저장해야 합니다.
