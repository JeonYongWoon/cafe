# 메인 오케스트라 에이전트 규칙 (Orchestrator Rules)

본 문서는 프로젝트의 전체 작업을 조율하고, 이슈 위험 단계를 분류하며, 오류 반복 방지를 위한 안전 정지(Safety Stop) 루프를 통제하는 메인 오케스트라 에이전트의 운영 규칙입니다.

## 1. 이슈 위험 단계 분류 정의

메인 오케스트라는 사용자의 요구사항을 분석하여 해당 이슈의 위험 등급을 반드시 사전에 정의해야 합니다.

- **고위험 (High Risk)**:
  - **대상**: 포인트 충전/차감 등 돈 관련 트랜잭션, DB 스키마 수정(DDL), 도메인 간 결합(비동기 이벤트 연동), 다중 스레드 동시성 처리가 요구되는 기능.
  - **조치**: 단순 Mock 테스트 검증을 전면 금지하며, 실제 로컬/인메모리 DB(H2) 또는 실제 인프라 환경과 유사한 **통합 테스트(Integration Test)**를 빌드 검증 단계에 강제합니다.
- **중위험 (Medium Risk)**:
  - **대상**: 새로운 API 생성, 비즈니스 핵심 로직 수정, 신규 도메인 엔티티 설계.
  - **조치**: JPA 리포지토리 슬라이스 테스트 및 컨트롤러 슬라이스 테스트(`@WebMvcTest`, `@DataJpaTest` 등)를 의무화합니다.
- **저위험 (Low Risk)**:
  - **대상**: 단순 조회(GET) API 추가, 오타 수정, 로그 메시지 개선, 단순 DTO 필드 추가.
  - **조치**: JUnit 기반 Mocking 단위 테스트 수준의 신속한 검증을 적용합니다.

## 2. 리뷰 거절 재시도 루프 및 안전 정지 (Safety Stop) 규칙

- **루프 정의**:
  - 구현 에이전트의 코드 제출 후 리뷰 에이전트가 거절(Reject) 처리를 하면, 구현 에이전트는 피드백을 수용하여 코드를 보완하고 재제출할 수 있습니다.
- **횟수 제약 및 제한**:
  - 동일한 이슈 혹은 PR 세션에서 허용되는 최대 거절 횟수는 **최대 2회**입니다.
  - 만약 리뷰 에이전트에 의해 **3번째 거절(거절 횟수 누적 3회)**이 발생하면, 오케스트라는 즉시 작업을 중단(안전 정지)하고 사용자 개입을 요청해야 합니다.
- **안전 정지 발생 시 조치**:
  - 오케스트라는 즉시 프로젝트 루트 또는 `docs/` 디렉토리에 [safety_stop_template.md](file:///Users/t2026-m0045/Downloads/cafe/docs/safety_stop_template.md) 규격을 준수하는 `docs/safety_stop_report.md` 파일을 생성해야 합니다.
  - 리포트 작성이 완료되면 에이전트는 추천 대안을 대화방에 제시하고 모든 자동 도구 사용을 정지합니다.
