# k6 부하 테스트 가이드 (LOAD_TEST)

본 문서는 커피숍 주문 시스템(jeonscafe)의 성능 및 안정성을 점검하기 위한 k6 부하 테스트 스크립트 작성 규격, 실행 방법 및 테스트 결과를 기록하는 문서입니다.

## 1. 부하 테스트 개요

- **테스트 도구**: k6
- **테스트 대상 API**:
  - GET /menus (전체 메뉴 목록 조회 API)
  - GET /menus/popular (7일 기준 인기 메뉴 목록 조회 API)
- **부하 설정 (Options)**:
  - 가상 사용자(VUs): 10명
  - 테스트 지속 시간: 30초
- **성능 임계치 (Thresholds)**:
  - 전체 요청 중 실패율 1% 미만 (http_req_failed < 0.01)
  - 상위 95% 요청의 응답 속도 200ms 이내 (http_req_duration p(95) < 200)

## 2. 테스트 환경 구성 및 실행 방법

### 로컬 환경에서 실행

1. **k6 설치**:
   터미널에서 Homebrew를 통해 k6를 설치합니다.
   ```bash
   brew install k6
   ```

2. **애플리케이션 구동**:
   Spring Boot 서버를 백그라운드 혹은 IDE에서 실행합니다.
   ```bash
   ./gradlew bootRun
   ```

3. **부하 테스트 수행**:
   프로젝트 루트 경로(/Users/t2026-m0045/Downloads/cafe)에서 아래 명령어를 실행합니다.
   ```bash
   k6 run k6/load-test.js
   ```

### 도커 다중 인스턴스 환경에서 실행

1. **도커 컴포즈 구동**:
   프로젝트 루트에서 Docker Compose를 사용해 다중 WAS 및 Nginx 로드 밸런서를 띄웁니다.
   ```bash
   docker-compose up -d
   ```

2. **부하 테스트 수행**:
   k6 실행 환경 변수로 로드 밸런서 주소(http://localhost)를 주입하여 실행합니다.
   ```bash
   k6 run --env BASE_URL=http://localhost k6/load-test.js
   ```

### GitHub Actions CI 환경에서 실행

- main 브랜치에 코드가 push되거나, GitHub Repository의 Actions 탭에서 **k6 Load Test Workflow**를 선택해 수동으로 실행(workflow_dispatch)할 수 있습니다.
- 워크플로 실행 완료 후 빌드 단계 요약 화면에서 애플리케이션의 콘솔 로그가 누적된 app.log를 아티팩트로 다운로드할 수 있습니다.
