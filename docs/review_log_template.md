# 에이전트 리뷰 및 검증 로그 템플릿

본 문서는 리뷰 에이전트가 검증 결과를 바탕으로 승인/거절 여부를 결정하고 작성하는 로그 템플릿 파일 예시입니다. 모든 검증 및 리뷰 로그는 `docs/review_logs/YYYYMMDD_HHMMSS_review.md` 경로에 본 포맷을 준수하여 기록되어야 합니다.

## 1. 세션 기본 정보
- **리뷰 일시**: YYYY-MM-DD HH:MM:SS
- **작업명**: (예: 포인트 충전 API 구현 검증 및 리뷰)
- **대상 브랜치 / 변경 사항**: (예: feature/charge-point)
- **식별된 이슈 위험도**: 고위험 (High Risk) / 중위험 (Medium Risk) / 저위험 (Low Risk)

## 2. 에이전트 운영 통계
- **검증 단계 사용된 총 에이전트 수**: N 명
- **동작 에이전트 종류**: (예: Implement, Verify, QA, Review Agent)

## 3. 사용자 실수 및 피드백 기록
- **사용자 실수 내역**:
  - (예: 요구사항 분석 단계에서 PointHistory 기록 요구사항을 누락하여 코드를 잘못 제안함)
  - (예: Controller에서 RESTful API 설계 위반 사항 발생)
- **보완 사항 및 조치 조언**:
  - (예: 요구사항 체크리스트를 더 철저히 검증하고, RESTful URI 규칙을 리마인드할 것)

## 4. 위험도별 테스팅 검증 여부
- **테스트 수준**: 통합 테스트 (Integration Test) / 슬라이스 테스트 (Slice Test) / 단위 테스트 (Unit Test)
- **실제 DB 연동 여부**: (예: 고위험 등급에 따른 H2 DB 연동 검증 성공)

## 5. 최종 리뷰 결과
- **의사 결정**: 승인 (Approve) / 거절 (Reject)
- **사유 및 주요 피드백 (Pn Rule)**:
  - (예: P1 - 컴파일 에러 없음. P2 - Lombok 가이드라인 준수 확인. 최종 승인함.)
