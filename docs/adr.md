# 아키텍처 결정 기록 (Architecture Decision Records - ADR)

본 문서는 jeonscafe 프로젝트에서 결정된 아키텍처 설계 의사결정 사항들을 투명하게 추적하고 관리하기 위한 기록(adr 문서)입니다.

## [ADR-001] Lombok 애너테이션 사용 제한 및 부작용 방지
- **상태**: 승인됨 (Approved)
- **결정 배경**:
  - 클래스 레벨의 `@Setter`와 `@Data` 무분별한 사용은 객체의 불변성을 저해하고 JPA 양방향 순환 참조에 의한 StackOverflowError를 유발할 수 있습니다.
  - `@AllArgsConstructor`는 필드 선언 순서 변경 시 치명적인 버그를 야기할 수 있습니다.
- **의사 결정**:
  - 엔티티 수정은 단순 Setter가 아닌 비즈니스적 의미를 갖는 커스텀 메서드로 제한합니다.
  - `@Data` 사용을 금지하고 개별 애너테이션을 적용하며, 기본 생성자는 `@NoArgsConstructor(access = AccessLevel.PROTECTED)`로 고정합니다.
  - 생성자 사용 시 `@RequiredArgsConstructor` 또는 Builder 패턴을 권장합니다.

## [ADR-002] Bounded Context 간 의존성 및 식별자 참조
- **상태**: 승인됨 (Approved)
- **결정 배경**:
  - 도메인 간의 강한 결합(JPA 객체 직접 참조 등)은 도메인 경계를 허물고 순환 참조를 발생시킵니다.
- **의사 결정**:
  - 컨텍스트 간의 직접적인 클래스/엔티티 참조를 금지하고 식별자 직접 참조(ID 참조) 방식을 기본으로 합니다.
  - 타 도메인 서비스를 직접 주입받을 수 없으며 퍼사드(Facade) 계층이나 이벤트를 경유하도록 설계합니다.

## [ADR-003] 트랜잭션 경계 분리 및 비동기 이벤트 핸들링
- **상태**: 승인됨 (Approved)
- **결정 배경**:
  - 하나의 트랜잭션 범위 내에 다른 컨텍스트 로직이 묶여 있으면 서비스 간 장애 전파 및 동기식 대기 지연 문제가 발생합니다.
- **의사 결정**:
  - Spring Application Event를 사용한 비동기 리스너 기반 처리를 표준으로 삼습니다.
  - 물리적으로 강한 동기식 트랜잭션 격리가 요구될 경우에만 `Propagation.REQUIRES_NEW`로 격리하고 필요한 보상 트랜잭션을 설계합니다.

## [ADR-004] 이슈 위험 등급별 다층적 검증 전략 도입
- **상태**: 승인됨 (Approved)
- **결정 배경**:
  - 목(Mock) 단위 테스트에만 과도하게 의존하여 검증을 진행할 경우, 실제 데이터베이스 연결, 트랜잭션 커밋/롤백 상태, 그리고 프로덕션(실제 배포) 환경에서 예상치 못한 오류가 터질 위험성이 큽니다.
- **의사 결정**:
  - 돈/결제, DDL 변경, 동시성 작업 등 고위험(High Risk) 업무에 대해서는 H2 등 인메모리 실제 DB 환경을 연동하는 통합 테스트(`@SpringBootTest`) 작성을 의무화합니다.
  - 중위험(Medium Risk) 업무는 계층별 슬라이스 테스트를 활용하여 Mocking을 제한적으로 사용하고, 저위험(Low Risk) 업무에 한해 순수 단위 테스트를 허용합니다.
